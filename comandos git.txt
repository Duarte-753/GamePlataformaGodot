git add . (pega todos os arquivos alterados da pasta raiz)

git status (verifica o status para fazer commit)

git commit -m "descrição do commit" (faz a descrição do commit que irá montar dos arquivos alterados)

git push origin master (faz o commit no brench master)

git pull origin master (puxa as alterções do brench escolhida)

git pull origin (puxa todas as alteraçoes e brench do projeto)

git branch (puxa o nome de todas as brench do projeto)

git checkout <nome-do-branch> (troca para a brench desejada se eu colocar -b na frente do checkout ele vai criar uma brench e selecionar ela)

git merge <nome-do-branch> posso usar o --squash e depois colocar o nome em um commit tipo -> git commit -m "merge julio"
(preciso estar em uma brench ex: estou na master, sendo assim quero puxar as alterações de uma brench
alterações Julio para master. ex: git merge <nome-do-branch> --> git merge julio vai juntar julio para master)
